it3191-example-urlrewrite

This is a simple example of urlrewrite using tuckey's urlrewrite filter.

I use it for the tidyness of my resources and URL management. The benefit is that I do not have to modify all my url mappings even if my resources name change, for example: myOldPage.html to myNewPage.html. The only thing I need to change is the URL Rewrite declaration in urlrewrite.xml.

From security point of view, I also use it to serve as a base for my resource restriction and access control. I can define URL rewrite as /admin/* and /public/* to seperate the accessibility of different users. This however require another filter to do the authentication & authorization.

There are tons of other useful feature like browser detection and conditional mappings. You should look at the official website for more example.

Additional Advise

Make sure you plan your url rewrite mappings well before you deploy. Especially general resources like css and js. Usually you don't want to rewrite or restrict access to them (or else they can't load!). Accidentally including them into url rewrite will render application misbehave.

Installation Guide

You should follow the 3 steps instruction at http://tuckey.org/urlrewrite/.

Things to take care of:

  1. Include the urlrewrite jar to WEB-INF/lib
  2. Declare filter mappings in web.xml
  3. Declare your rewrite rules in urlrewrite.xml. You should create your own urlrewrite.xml or make a copy from https://urlrewritefilter.googlecode.com/svn/trunk/src/doc/manual/4.0/urlrewrite.xml
